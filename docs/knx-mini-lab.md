# KNX ミニラボ設計指針

KNX ミニラボは、実際の住宅や建物を改造せずに KNX システムの設計と統合を学ぶための小規模な実験環境です。ここでは、最小構成から発展的な応用までの設計方針をまとめます。

## 目的

*   KNXのライン・エリア構造やグループアドレスの設計方法を理解する
*   IPインターフェースを用いてKNXと他のスマートホーム機器を連携させる
*   実際の施工を行わずに机上で完結するテスト環境を構築する

## 構成例

ミニラボの基本構成は以下の通りです。

*   KNX 電源：バス電圧を供給するための装置
*   KNX IP インターフェース：KNX バスと外部ネットワークの橋渡しをするデバイス
*   スイッチ（センサー）：ユーザー操作を KNX バスに伝える
*   アクチュエータ：照明や機器を制御するための出力デバイス
*   ランプや小型負荷：制御対象として机上で動作を確認する

## トポロジ

ラインとエリアの概念を適用し、アドレス体系を整理します。ミニラボでは単一ライン構成で十分ですが、グループアドレスの割り当てを明確にすることで、大規模システムに移行した際のスケーラビリティを意識します。

*   エリア 1、ライン 1（1.1）の構成とする
*   グループアドレスは 3 レベル方式（メイン/ミドル/サブ）の番号体系を使用
*   センサーとアクチュエータ間の関係を定義し、シーン制御やフィードバックを設計

## IP 連携

KNX IPインターフェースを用いることで、外部システムとKNXを統合できます。具体的には以下の要素を検討します。

*   KNXnet/IP トンネリングによる ETS 設定とアクセス
*   Hue ブリッジや Oura API との連携を行う中間サーバー（Raspberry Pi など）の設置
*   イベント駆動で KNX から Hue シーンを切り替えるフローと、逆方向の状態反映
*   セキュリティ確保のためのネットワーク分離と認証設計

## 発展的なアイデア

*   エネルギー計測ユニットを追加し、消費電力に応じてシーンや制御ロジックを動的に変更する
*   AIスケジューリングや学習アルゴリズムを取り入れ、生活パターンに応じた自動化を実現する
*   ウェブダッシュボードを作成し、グループアドレスやシーンを視覚的に管理できる UI を実装

## 注意点

*   壁内配線への接続は行わず、机上で独立した環境として構築する
*   安全のため、電源の取り扱いや負荷機器の接続はメーカーの仕様に従う
*   市販デバイスや中古品を活用してコストを抑える
