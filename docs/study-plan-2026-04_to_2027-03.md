# 2026年4月〜2027年3月 年間計画

このドキュメントは、KNX ミニラボの構築と、2026年11月の応用情報技術者試験、2027年2月のネットワークスペシャリスト試験に向けた年間計画を、月ごとに整理したものです。

## この計画の目的

  -   KNX を軸にスマートホーム設計の理解を深める
  -   自宅で遊べる形でミニラボを構築する
  -   資格勉強を通じてネットワークとセキュリティの土台を固める
  -   ドキュメントと実装を積み上げてポートフォリオにする

## 2026年4月

テーマは現状把握と年間設計です。

  -   応用情報とネットワークスペシャリストの出題範囲を把握する
  -   年間の学習時間の目安を決める
  -   自宅のネットワーク構成を図に起こす
  -   GitHub リポジトリのドキュメント構成を整える
  -   KNX ミニラボの目的、対象範囲、やらないことを明文化する

月末ゴール：1年間の方針が言語化され、現状の自宅ネットワーク図がある状態

## 2026年5月

テーマは応用情報の基礎固めとネットワークの整理です。

  -   TCP/IP、OSI 参照モデル、IP アドレス、サブネットの復習を進める
  -   DNS、HTTP、HTTPS、NAT の基本動作を整理する
  -   スマートホーム用ネットワーク分離の案を考える
  -   KNX ミニラボで使う機器候補を洗い出す
  -   ETS の情報収集と導入準備を進める

月末ゴール：基礎ネットワークの理解が一巡し、機器選定の方向性が見えている状態

## 2026年6月

テーマはプロトコル理解とラボ準備です。

  -   TCP と UDP の使い分けを整理する
  -   マルチキャスト、ルーティング、VLAN の基本を学ぶ
  -   KNXnet/IP の概要を理解する
  -   Hue API の基本的な操作を確認する
  -   KNX ミニラボの購入候補と予算案をまとめる

月末ゴール：KNX IP と既存スマートホームの接続イメージを説明できる状態

## 2026年7月

テーマは応用情報の問題演習開始とラボ着手です。

  -   応用情報の午前問題を解き始める
  -   頻出分野の弱点を記録する
  -   ETS を導入して仮想プロジェクトを作る
  -   KNX のグループアドレス設計の考え方を学ぶ
  -   ミニラボ用の機器を購入し始める

月末ゴール：応用情報の午前問題に慣れ始め、ETS 上で最低限の構成を触れる状態

## 2026年8月

テーマは応用情報の午後対策と実機セットアップです。

  -   応用情報の午後問題の形式に慣れる
  -   ネットワーク分野を重点的に解く
  -   KNX 電源、IP インターフェース、スイッチ、アクチュエータの机上配線を行う
  -   ETS で簡単な制御を設定する
  -   学習メモと実験ログを GitHub に残す

月末ゴール：KNX ミニラボが机上で立ち上がり、応用情報の午後問題に着手できている状態

## 2026年9月

テーマは応用情報の実戦化と IP 連携の入口です。

  -   応用情報の過去問を時間を測って解く
  -   記述の書き方を改善する
  -   KNXnet/IP でイベントを取得する方法を調べる
  -   Hue との連携プロトタイプの構成を考える
  -   自宅スマートホーム全体のアーキテクチャ図を作る

月末ゴール：応用情報が合格ラインに近づき、ラボの全体アーキテクチャ図ができている状態

## 2026年10月

テーマは応用情報の総仕上げです。

  -   応用情報の午前と午後を通しで解く
  -   弱点分野を絞って復習する
  -   ラボのネットワーク設計とセキュリティ方針を整理する
  -   KNX から Hue への基本連携フローを文章で整理する
  -   試験直前に増やすものと増やさないものを決める

月末ゴール：応用情報の合格ラインが安定し、ラボ側も次の実装テーマが明確になっている状態

## 2026年11月

テーマは応用情報本番と NW への切り替えです。

  -   応用情報技術者試験を受験する
  -   試験後に手応えと弱点を記録する
  -   すぐにネットワークスペシャリスト対策へ切り替える
  -   NW の出題傾向と午後記述の型を確認する
  -   ラボでは通信方式や構成要素を改めて整理する

月末ゴール：応用情報が完了し、NW 対策モードに移行している状態

## 2026年12月

テーマは NW の基礎固めと設計目線の強化です。

  -   NW の午前 II・午後対策を始める
  -   VLAN、冗長化、ロードバランシング、VPN の基礎を整理する
  -   設計問題の読み方と回答の型を学ぶ
  -   ミニラボでネットワーク分離や役割分担を見直す
  -   KNX、Hue、中間サーバーの責務を整理する

月末ゴール：NW の設計問題に対する読み方が見え始めている状態

## 2027年1月

テーマは NW 本番対策とラボの統合設計です。

  -   NW の午後問題を本格的に解く
  -   記述の精度を上げる
  -   KNX と Hue を連携する最小プロトタイプを作る
  -   認証、通信経路、分離方針を図にまとめる
  -   ポートフォリオとして見せられる資料の骨子を作る

月末ゴール：NW の午後問題にある程度対応でき、ラボの統合設計が見える状態

## 2027年2月

テーマは NW 本番です。

  -   ネットワークスペシャリスト試験を受験する
  -   新しい教材には手を広げず、過去に整理した内容を中心に復習する
  -   ラボでは既存構成の安定化とログ整理を行う
  -   試験終了後に学習内容とラボ内容の接続を振り返る

月末ゴール：試験を終え、今後のラボ強化ポイントが見えている状態

## 2027年3月

テーマは成果の整理と次年度の計画作成です。

  -   2026年度の学習内容と実装内容を振り返る
  -   GitHub 上のドキュメントを整理する
  -   ポートフォリオ用にプロジェクト概要を再構成する
  -   次年度に取り組むテーマを決める
  -   必要であればセキュリティ系や IoT 系の次の資格候補を検討する

月末ゴール：このプロジェクトの成果が第三者に説明できる形で整理されている状態

## 補足メモ

  -   応用情報は通過点として扱い、ネットワークと設計に関係する分野を重視する
  -   ネットワークスペシャリストは本命として扱い、設計理由を言語化する訓練を重視する
  -   机上ラボは施工の代替ではなく、アーキテクチャ理解と統合設計の実践の場と位置づける
  -   毎月末に、学習、実装、ドキュメントの3点で振り返りを行う
