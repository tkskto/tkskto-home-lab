# タスク

このドキュメントでは、KNX ミニラボ構築に関連するタスクや今後の学習・研究メモを整理します。進捗管理やアイデアの整理に活用してください。

## 現在のタスク

*   ETS デモ版をインストールし、基本的なプロジェクトを作成する
*   KNX 電源・IP インターフェース・スイッチ・アクチュエータを購入し、机上にて配線を行う
*   グループアドレス設計を行い、簡単なシーン制御を ETS 上で設定する
*   中間サーバー（Raspberry Pi または Mac）を準備し、KNXnet/IP 経由でイベントを監視するプログラムを実装する
*   Hue API を用いたシーン切替と、Oura API からのデータ取得を組み合わせた連携プロトタイプを作成する
*   ウェブダッシュボードの設計およびプロトタイプ作成を検討する

## 予定しているタスク

*   エネルギー計測ユニットの導入とデータ可視化手法の検討
*   AI モデルを用いた自動シーン調整アルゴリズムの検証
*   セキュリティ面の調査とネットワーク分離設計の詳細化
*   KNX 以外のプロトコル（Zigbee、Matter など）との連携可能性の調査

## 完了済みタスク

*   リポジトリの作成と基本的な README の追加
*   学習計画の立案とロードマップの策定

## 研究メモ

*   KNX は分散型アーキテクチャであり、中央サーバーに依存しない構成が特徴
*   グループアドレスは機能単位で設計し、論理的なまとまりを意識する
*   IP 連携においては、トンネリング（エンドポイント間直接接続）とルーティング（マルチキャスト）の違いを理解して選択する
*   Hue API は REST ベースであり、ユーザー認証とブリッジへの登録が必要
*   Oura API では OAuth 認証が必要で、取得できるデータの範囲を確認する
